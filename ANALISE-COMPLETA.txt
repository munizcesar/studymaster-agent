RESPOSTA 3: Por que pode não estar funcionando em Desktop?
═════════════════════════════════════════════════════════════════════════════

PROBLEMA 1 [CRÍTICO] - CSS FALTANDO
┌─────────────────────────────────────────────────────────────────────────┐
│ A classe .option-btn.selected NÃO TEM ESTILOS CSS!                     │
│                                                                         │
│ No Fullscreen Mode (.qf-option.selected):                            │
│   ✓ border-color: var(--color-primary);                              │
│   ✓ background: oklch(...);                                           │
│   ✓ .qf-option-letter tem estilos                                    │
│                                                                         │
│ Na página normal (.option-btn.selected):                              │
│   ❌ NADA! Não há regra CSS definida                                 │
│   ❌ Resulta em: visual não diferente da não-selecionada             │
│                                                                         │
│ IMPACTO: Usuário clica em opção, mas ela não aparenta selecionada    │
└─────────────────────────────────────────────────────────────────────────┘

PROBLEMA 2 [ALTO] - MÚLTIPLOS EVENT LISTENERS
┌─────────────────────────────────────────────────────────────────────────┐
│ Linhas 2960-2968:                                                       │
│   if (!qNavState.results[idx]) {                                        │
│     opts.forEach(btn => {                                              │
│       btn.addEventListener('click', () => { ... });                    │
│     });                                                                 │
│   }                                                                     │
│                                                                         │
│ Problema: Se renderAllQuestionsDesktop() é chamado múltiplas vezes    │
│   → A condição if (!qNavState.results[idx]) permanece verdadeira      │
│   → addEventListener é ADICIONADO novamente                            │
│   → Cada botão tem 2, 3, 4... listeners do mesmo tipo                 │
│   → Cada clique ativa TODOS os listeners anteriores                   │
│                                                                         │
│ Cenário: Resize mobile→desktop, ou navegação rápida                  │
│   → Múltiplos listeners acumulam                                       │
│   → Comportamento inconsistente                                        │
│                                                                         │
│ IMPACTO: Cliques erráticos, seleções múltiplas, desempenho ruim       │
└─────────────────────────────────────────────────────────────────────────┘

PROBLEMA 3 [MÉDIO] - ESTADO NÃO RESETADO NO RESIZE
┌─────────────────────────────────────────────────────────────────────────┐
│ Linhas 3559-3584 (resize event):                                        │
│   if (currentMode !== lastViewportMode) {                              │
│     if (isDesktop) {                                                   │
│       renderAllQuestionsDesktop(state.generatedQuestions);             │
│     }                                                                   │
│   }                                                                     │
│                                                                         │
│ Problema: Antes de renderizar, NÃO reseta:                            │
│   ❌ qNavState.selected                                                │
│   ❌ qNavState.results                                                 │
│   ❌ qNavState.currentIdx                                              │
│                                                                         │
│ Cenário: Usuário em mobile (respondendo questão 2), faz resize         │
│   → renderSingleQuestion() renderizou questão 2                        │
│   → qNavState tem estado parcial                                       │
│   → renderAllQuestionsDesktop() usa qNavState sujo                     │
│   → Questões mostram marcações erradas                                 │
│                                                                         │
│ IMPACTO: Após resize, opções aparecem pré-marcadas incorretamente     │
└─────────────────────────────────────────────────────────────────────────┘

PROBLEMA 4 [BAIXO] - FALTA VALIDAÇÃO EXTRA
┌─────────────────────────────────────────────────────────────────────────┐
│ Cliques podem ocorrer após responder se houver race condition           │
│ Sugestão: Adicionar guard if (qNavState.results[idx]) return;          │
│           dentro do addEventListener.click                             │
└─────────────────────────────────────────────────────────────────────────┘


RESPOSTA 4: Qual seria a melhor forma de corrigir?
═════════════════════════════════════════════════════════════════════════════

CORREÇÃO 1 [CRÍTICO] - ADICIONAR CSS FALTANDO
─────────────────────────────────────────────────────────────────────────
Local: Logo após linha 487 (após .option-btn.wrong)

Adicionar:
  .option-btn.selected {
    border-color: var(--color-primary);
    background: oklch(from var(--color-primary) l c h / 0.05);
  }
  .option-btn.selected .option-key {
    border-color: var(--color-primary);
    color: var(--color-primary);
  }

Referência: Use .qf-option.selected (linhas 879-888) como modelo

IMPACTO: ✓ Opções marcadas ficarão visualmente distintas


CORREÇÃO 2 [ALTO] - EVITAR MÚLTIPLOS LISTENERS
─────────────────────────────────────────────────────────────────────────
Local: Linhas 2954-2979 em renderAllQuestionsDesktop()

Método 1: Usar cloning para limpar listeners antigos
  opts.forEach(btn => {
    const newBtn = btn.cloneNode(true);  // ← Remove listeners
    btn.replaceWith(newBtn);
    newBtn.addEventListener('click', () => { ... });
  });

Método 2: Remover evento antes de adicionar
  btn.removeEventListener('click', null);
  btn.addEventListener('click', handler);

Recomendação: Use o método 1 (mais seguro)

IMPACTO: ✓ Cada clique ativa apenas 1 listener


CORREÇÃO 3 [MÉDIO] - RESETAR ESTADO ANTES DE RENDERIZAR
─────────────────────────────────────────────────────────────────────────
Local 1: Na função renderQuestions() (linhas 2872-2876)
  Adicionar LOGO APÓS inicializar qNavState:
  
  qNavState = { ... };
  document.getElementById('questions-list').innerHTML = '';  // ← Limpa

Local 2: No resize event listener (linhas 3573-3584)
  Adicionar ANTES de renderizar:
  
  qNavState = {
    currentIdx: 0,
    selected: new Array(state.generatedQuestions.length).fill(null),
    results: new Array(state.generatedQuestions.length).fill(null),
  };

IMPACTO: ✓ Resize não deixa estado sujo
         ✓ Navegação não carrega estado antigo


CORREÇÃO 4 [MENOR] - ADICIONAR GUARDS EXTRAS
─────────────────────────────────────────────────────────────────────────
Dentro de cada addEventListener.click:
  
  newBtn.addEventListener('click', () => {
    if (qNavState.results[idx] !== null) return;  // ← Guard
    // ... resto do código
  });

IMPACTO: ✓ Previne comportamento após responder
